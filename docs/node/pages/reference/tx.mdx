# @tevm/tx

The `@tevm/tx` package is Tevm's transaction facade backed by `@evmts/zevm/tx`. It exports typed Ethereum transaction constructors, helpers, type guards, and Tevm's impersonated transaction helper for local execution and fork testing.

> **Generated API Documentation**: View the full API documentation in the [evmts/tevm-monorepo/packages/tx/docs](https://github.com/evmts/tevm-monorepo/tree/main/packages/tx/docs) folder.

## Installation

```bash
npm install @tevm/tx
```

## API Reference

### Transaction Types

- [LegacyTransaction](https://github.com/evmts/tevm-monorepo/tree/main/packages/tx/docs/classes/LegacyTransaction.md) - Legacy pre-EIP-2718 transactions.
- [AccessListEIP2930Transaction](https://github.com/evmts/tevm-monorepo/tree/main/packages/tx/docs/classes/AccessListEIP2930Transaction.md) - EIP-2930 access-list transactions.
- [FeeMarketEIP1559Transaction](https://github.com/evmts/tevm-monorepo/tree/main/packages/tx/docs/classes/FeeMarketEIP1559Transaction.md) - EIP-1559 fee-market transactions.
- [BlobEIP4844Transaction](https://github.com/evmts/tevm-monorepo/tree/main/packages/tx/docs/classes/BlobEIP4844Transaction.md) - EIP-4844 blob transactions.
- `EOACodeEIP7702Transaction` - EIP-7702 EOA code transactions.
- [ImpersonatedTx](https://github.com/evmts/tevm-monorepo/tree/main/packages/tx/docs/interfaces/ImpersonatedTx.md) - Tevm-specific transaction shape for executing as an address without a private key.

### Helpers

- `TransactionFactory` - Alias for ZEVM's `createTx` helper.
- `createTxFromRLP` - Decode a serialized transaction.
- `createTxFromBlockBodyData` - Create a transaction from block body data.
- `createImpersonatedTx` - Create an unsigned EIP-1559-shaped transaction that executes as `impersonatedAddress`.
- `createEOACodeEIP7702Tx`, `createEOACodeEIP7702TxFromBytesArray`, and `createEOACodeEIP7702TxFromRLP` - EIP-7702 helpers.
- `isLegacyTx`, `isAccessListEIP2930Tx`, `isFeeMarketEIP1559Tx`, `isBlobEIP4844Tx`, and `isEOACodeEIP7702Tx` - Transaction type guards.

## Creating Transactions

```typescript
import {
  TransactionFactory,
  createImpersonatedTx,
  createTxFromRLP,
  isBlobEIP4844Tx,
  isEOACodeEIP7702Tx,
} from '@tevm/tx'
import { createAddress } from '@tevm/address'

const tx = TransactionFactory({
  nonce: 0n,
  gasLimit: 21000n,
  maxFeePerGas: 20_000_000_000n,
  maxPriorityFeePerGas: 2_000_000_000n,
  to: createAddress('0x1234567890123456789012345678901234567890'),
  value: 1_000_000_000_000_000_000n,
})

const decoded = createTxFromRLP(tx.serialize())

if (isBlobEIP4844Tx(decoded) || isEOACodeEIP7702Tx(decoded)) {
  // Handle modern typed transaction fields.
}
```

## Impersonated Transactions

Impersonated transactions are a Tevm convenience for simulation, fork testing, and local development. They let the VM execute as an address without requiring a signature from that address.

```typescript
import { createImpersonatedTx } from '@tevm/tx'
import { createAddress } from '@tevm/address'

const tx = createImpersonatedTx({
  impersonatedAddress: createAddress('0xf39fd6e51aad88f6f4ce6ab8827279cfffb92266'),
  to: createAddress('0x70997970c51812dc3a010c7d01b50e0d17dc79c8'),
  value: 1_000_000_000_000_000_000n,
  gasLimit: 21_000n,
  maxFeePerGas: 20_000_000_000n,
  maxPriorityFeePerGas: 2_000_000_000n,
  nonce: 0n,
})

const vm = await node.getVm()
const result = await vm.runTx({ tx, skipBalance: true, skipNonce: true })
```

For app-level impersonation, prefer the JSON-RPC or MemoryClient helpers such as `anvil_impersonateAccount`, `tevm_impersonateAccount`, and `setImpersonatedAccount`.

## Transaction Type Coverage

The package supports:

- Legacy transactions
- EIP-2930 access-list transactions
- EIP-1559 fee-market transactions
- EIP-4844 blob transactions
- EIP-7702 EOA code transactions
- Tevm impersonated transactions

## Common Methods

Typed transaction instances expose the standard transaction helpers from the underlying ZEVM transaction implementation:

- `hash()` - Get the transaction hash.
- `getBaseFee()` - Get intrinsic gas.
- `getDataFee()` - Get calldata or blob-related gas cost.
- `getUpfrontCost()` - Get total required account balance.
- `isSigned()` - Check whether the transaction is signed.
- `serialize()` - Return the encoded transaction.
- `toJSON()` - Return the JSON representation.
